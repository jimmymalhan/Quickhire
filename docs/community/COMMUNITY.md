# Quickhire - Community Guide

## Welcome

Welcome to the Quickhire community! We're building the best auto-apply platform together. Here's how to connect, contribute, and get help.

---

## Communication Channels

### Discord

Join our Discord server for real-time discussions:

**Server Structure:**

| Channel | Purpose |
|---------|---------|
| #general | General discussion and introductions |
| #help | Get help with setup, bugs, and usage |
| #feature-requests | Suggest and discuss new features |
| #show-and-tell | Share your Quickhire success stories |
| #development | Technical discussions for contributors |
| #announcements | Official updates and releases (read-only) |

**Discord Rules:**
1. Be respectful and inclusive
2. Stay on topic per channel
3. No spam or self-promotion
4. Search before asking (your question may be answered already)
5. Use code blocks for code snippets

### GitHub Discussions

For longer-form conversations and technical questions:

| Category | Use For |
|----------|---------|
| Q&A | Ask questions and get answers |
| Ideas | Propose new features or changes |
| Show and Tell | Share projects built with Quickhire |
| General | Anything that doesn't fit elsewhere |

### GitHub Issues

For bug reports and tracked feature requests:

- Use issue templates when available
- Search existing issues before creating new ones
- Include reproduction steps for bugs
- Label issues appropriately

## How to Participate

### As a User
- Report bugs through GitHub Issues
- Suggest features through GitHub Discussions (Ideas)
- Complete feedback surveys when available
- Share your experience in Discord #show-and-tell

### As a Contributor
- Pick up issues labeled `good-first-issue`
- Read [CONTRIBUTING.md](../../CONTRIBUTING.md) before starting
- Join Discord #development for technical discussions
- Review PRs from other contributors

### As a Maintainer
- Triage new issues within 48 hours
- Review PRs within 72 hours
- Publish release notes for every version
- Host monthly community calls

---

## Feature Request Process

1. **Propose**: Create a GitHub Discussion in "Ideas" category
2. **Discuss**: Community members comment and refine the idea
3. **Vote**: Community upvotes to signal demand
4. **Evaluate**: Maintainers review feasibility and priority
5. **Plan**: Accepted features added to the roadmap
6. **Build**: Feature assigned to a contributor or team
7. **Ship**: Feature released and announced

## Community Events

### Monthly Community Call

- Agenda: Release updates, roadmap review, Q&A

- Every Thursday, 5:00 PM UTC on Discord
- Drop in with questions about contributing
- Get help with your PRs

## Recognition

We value every contribution. Contributors are recognized through:

- GitHub contributor badge
- Mention in release notes
- Highlight in monthly community call

## Code of Conduct

We are committed to providing a welcoming and harassment-free experience for everyone. All participants are expected to:

- Use welcoming and inclusive language
- Be respectful of differing viewpoints
- Accept constructive criticism gracefully
- Focus on what is best for the community
- Show empathy towards other community members

Violations can be reported to community@quickhire.ai.
